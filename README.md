0. Сбор данных производился через парсер `kolesa_parser.py`. Все нужные для дальнейших пунктов данные лежат в `data/raw`.

Итого: `clean_df` => `merge_parsed.ipynb` -> `prepare_raw_data.ipynb` -> `split_datasets.ipynb` => `df_table.csv` & `df_images.ipynb`

## Логирование экспериментов (W&B)

Все эксперименты и по таблице, и по картинкам логируются в Weights & Biases
Каждая модель обучается отдельным запуском со своим именем (например ResNet18_finetune_brand), поэтому их удобно сравнивать между собой

В каждый запуск пишется:
- параметры обучения (lr, число эпох, размер батча, seed) и архитектура модели
- метрики: для картинок - по эпохам (loss, accuracy), для таблицы - loss и MAE/RMSE/R2
- обученная модель как артефакт (файл с весами)
- данные, на которых училась модель, тоже как артефакт (kolesa-data для картинок, kolesa-tabular для таблицы)

Это дает то, что по сохранённой модели можно получить те же предсказания, не обучая заново. Скачиваем веса из W&B, грузим в модель, готово. А раз рядом лежат и данные, эксперимент можно повторить целиком, имея доступ только к W&B

### Как включить
В CFG стоит флаг wandb = True. При первом запуске wandb.login() попросит API ключ (взять на https://wandb.ai/authorize). Если поставить wandb = False обучение идёт как обычно, но без записи в облако
